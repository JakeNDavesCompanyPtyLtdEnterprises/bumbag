# Theming

Fannypack can be extensively themed to meet your requirements. You can either **override the default Fannypack theme** or **create your own custom theme** from scratch. If you like the default theme and want to keep it simple, we recommend you just override the default.

## Overriding the default theme (recommended)

The `<ThemeProvider>` component accepts `theme` as a prop, where you can specify overrides to the default Fannypack theme. The schema of the `theme` object can be seen [here](#theme-config-schema).

[Click here](https://github.com/fannypackui/fannypack/blob/master/src/themes/default.js) to see the default theme.

### Example

```jsx
import { ThemeProvider, css } from 'fannypack';

const theme = {
  global: {
    base: css`
      font-family: 'Comic Sans MS'
    `
  },
  palette: {
    primary: 'blue'
  },
  layout: {
    mobileBreakpoint: 520,
    tabletBreakpoint: 960
  },
  Button: {
    disabled: css`
      opacity: 0.2
    `
  },
  Text: css`
    font-weight: 300;
  `
}

const App = () => (
  <ThemeProvider theme={theme}>
    // ... your app
  </ThemeProvider>
);
```

## Creating your own theme (advanced)

If you decide to not use the default Fannypack theme and roll out your own, just add the `isStandalone` flag to `<ThemeProvider>`.

### Example

```jsx
import { ThemeProvider, defaultTheme } from 'fannypack';

const App = () => (
  <ThemeProvider isStandalone theme={defaultTheme()}>
    // ... your app
  </ThemeProvider>
);
```

#### LayoutThemeConfig

The layout theme config is used to manage breakpoints & spacing (like paddings & margins).

```jsx
{
  mobileBreakpoint: number,
  tabletBreakpoint: number,
  desktopBreakpoint: number,
  widescreenBreakpoint: number,
  fullHDBreakpoint: number,
  gapFactor: number,
  minorUnit: number,
  majorUnit: number
}
```

##### Default

```jsx
{
  mobileBreakpoint: 480,
  tabletBreakpoint: 768,
  desktopBreakpoint: 1024,
  widescreenBreakpoint: 1200,
  fullHDBreakpoint: 1440,
  gapFactor: 0.5,
  minorUnit: 4,
  majorUnit: 8
}
```
