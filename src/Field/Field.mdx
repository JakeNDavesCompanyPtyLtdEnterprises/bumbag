# Field

## Import

`import { Field } from 'fannypack'`

## Basic Usage

The `<Field>` component is a utility belt that wraps around form elements.

<Playground>
  <Field a11yId="username" label="Username">
    <Input />
  </Field>
</Playground>

## Descriptions

The description text is placed underneath the label.

<Playground>
  <Field a11yId="username" label="Username" description="Required for your fannypack">
    <Input />
  </Field>
</Playground>

## Hints

The hint text is placed underneath the field child.

<Playground>
  <Field a11yId="username" label="Username" hint="Must be awesome">
    <Input />
  </Field>
</Playground>

## Optional fields

To indicate a field as optional, add the `isOptional` prop.

<Playground>
  <Field a11yId="username" label="Username" isOptional>
    <Input />
  </Field>
</Playground>

## Required fields

Required fields don't show any visual feedback. Displaying 'optional' labels (as above) instead of a red astrix is more descriptive, less fearful, and accessible. However, adding the `isRequired` prop to the field will enable screen readers to identify required fields.

<Playground>
  <Field a11yId="username" label="Username" isRequired>
    <Input />
  </Field>
</Playground>
